# Task 5.3 — Profile Settings (Edit Profile, Change Password)

## Goal
Create a settings page where users can edit their profile and change their password.

## Prerequisites
- Wave 1 complete (auth, profiles)

## Steps

### 1. Settings Page
Create `app/dashboard/settings/page.tsx`:
- Server component — fetch current profile
- Auth guard

### 2. Profile Edit Form
Create `app/dashboard/settings/profile-form.tsx` (client component):
- Fields: full name, phone, bio (providers only), address
- Avatar upload placeholder (just show current avatar or initials, upload can be added later)
- "Save Changes" button
- Uses `updateProfile` action from `app/actions/profile.ts`
- Success/error feedback

### 3. Password Change Form
Create `app/dashboard/settings/password-form.tsx` (client component):
- Fields: new password, confirm new password
- Validation: passwords must match, minimum length
- Uses Supabase `updateUser({ password })` via a server action
- Add to `app/actions/auth.ts`:
  ```ts
  export async function changePassword(formData: FormData) {
    const supabase = await createClient()
    const newPassword = formData.get('password') as string
    const { error } = await supabase.auth.updateUser({ password: newPassword })
    if (error) return { error: error.message }
    return { success: true }
  }
  ```

### 4. Settings Layout
- Two sections: "Profile Information" and "Security"
- Each in its own card
- Both forms independently submittable

### 5. Design
- Dark cards for each section
- Gold save buttons
- Clear section separation
- Success toast/message on save
- Responsive single-column layout

### 6. Verify
```bash
npm run build
```

## Output Files
- `app/dashboard/settings/page.tsx`
- `app/dashboard/settings/profile-form.tsx`
- `app/dashboard/settings/password-form.tsx`
- `app/actions/auth.ts` (updated with changePassword)
