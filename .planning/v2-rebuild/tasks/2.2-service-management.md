# Task 2.2 — Service Management CRUD

## Goal
Create a service management page where providers can add, edit, and delete their detailing services (e.g., "Basic Wash $30", "Full Detail $150").

## Prerequisites
- Task 2.1 complete (provider onboarding, profile actions)

## Context
Each provider offers multiple services with a title, description, price, and duration. Customers will choose from these when booking.

## Steps

### 1. Service Server Actions
Create `app/actions/services.ts`:
```ts
'use server'
```
Actions:
- `getProviderServices(providerId?: string)` — fetch services for a provider (default: current user)
- `createService(formData: FormData)` — create new service (title, description, price, duration_minutes, is_active)
- `updateService(id: string, formData: FormData)` — update existing service
- `deleteService(id: string)` — soft delete or hard delete service
- `toggleServiceActive(id: string, isActive: boolean)` — toggle active/inactive

All actions should verify the current user is the owner of the service.

### 2. Services Dashboard Page
Create `app/dashboard/provider/services/page.tsx`:
- Server component — fetch provider's services
- Auth guard: redirect to `/login` if not authenticated
- Display list of services in cards
- "Add New Service" button at top
- Each service card shows: title, price, duration, active status, edit/delete buttons
- Empty state: "No services yet. Add your first service to start receiving bookings."

### 3. Add/Edit Service Dialog or Page
Create `app/dashboard/provider/services/service-form.tsx` (client component):
- Can be used for both create and edit
- Fields: title (required), description (textarea), price in dollars (required, number), duration in minutes (required, number select or input), is_active toggle
- Use a dialog/modal pattern OR inline form — keep it simple
- For simplicity, use a card form that appears above the list

### 4. Provider Dashboard Layout
Create `app/dashboard/provider/layout.tsx`:
- Sidebar or top navigation for provider dashboard sections
- Links: Overview, Services, Availability, Bookings (future), Stripe (future)
- Active link highlighting
- Use dark theme styling

### 5. Design
- Service cards with dark surface, subtle border
- Price displayed prominently
- Duration shown with clock icon
- Active/inactive badge (green/gray)
- Gold accent on "Add Service" button
- Use Lucide icons: Plus, Edit, Trash2, Clock, DollarSign

### 6. Verify
```bash
npm run build
```

## Output Files
- `app/actions/services.ts`
- `app/dashboard/provider/layout.tsx`
- `app/dashboard/provider/services/page.tsx`
- `app/dashboard/provider/services/service-form.tsx`
