# Task 3.4 — Provider Detail + Service Selection

## Goal
Enhance the provider profile page with service selection that leads to booking.

## Prerequisites
- Task 2.4 complete (provider profile page exists)
- Task 3.2 complete (provider API exists)

## Steps

### 1. Enhance Provider Profile Page
Update `app/provider/[id]/page.tsx`:
- Make service cards selectable/interactive
- Each service has a "Book Now" button that links to `/booking/[providerId]?service=[serviceId]`
- Add a sticky footer or sidebar CTA on mobile: "Book [Provider Name]"
- Show provider's service area on a small static map (optional, nice-to-have)

### 2. Service Selection Component
Create `components/providers/service-selector.tsx` (client component):
- Displays provider's services as selectable cards
- Selected service highlighted with gold border
- Shows price and duration prominently
- "Continue to Booking" button when a service is selected
- Links to the booking flow page

### 3. Reviews Enhancement
- Show average rating prominently at top
- Individual reviews with star display
- Sort by newest first
- Show "Verified Booking" badge on reviews

### 4. Design
- Clean provider profile layout
- Service cards are the main interactive element
- Gold highlight on selected/hovered service
- Clear CTA to book
- Responsive layout

### 5. Verify
```bash
npm run build
```

## Output Files
- `app/provider/[id]/page.tsx` (enhanced)
- `components/providers/service-selector.tsx`
