# Task 6.4 — Vercel Deployment + Doppler Integration

## Goal
Prepare the project for Vercel deployment with proper configuration.

## Prerequisites
- Waves 1-5 complete

## Steps

### 1. Vercel Configuration
The project is already set up at rinse-it.vercel.app. Ensure the following:

Create `vercel.json` (if needed for any custom config):
```json
{
  "$schema": "https://openapi.vercel.sh/vercel.json"
}
```
Note: Most configuration is handled by Vercel's Next.js auto-detection. Only add vercel.json if custom settings are needed.

### 2. Environment Variables Check
Ensure the following env vars are documented (they live in Doppler, synced to Vercel):
- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `SUPABASE_SERVICE_ROLE_KEY` (for admin operations)
- `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY`
- `STRIPE_SECRET_KEY`
- `STRIPE_WEBHOOK_SECRET`
- `NEXT_PUBLIC_MAPBOX_TOKEN`
- `RESEND_API_KEY`

Create a `.env.example` file documenting all required vars (with placeholder values).

### 3. Next.js Config for Production
Update `next.config.ts` if needed:
- Add image domains for Supabase storage (if using avatars)
- Any other production optimizations

### 4. Build Verification
```bash
npm run build
```
Must pass with zero errors and warnings.

### 5. Verify All Routes
Ensure all routes are properly compiled:
- Static pages: /, /login, /register
- Dynamic pages: /dashboard, /provider/[id], /booking/[providerId]
- API routes: /api/providers/nearby, /api/stripe/*
- Auth: /auth/callback

## Output Files
- `.env.example`
- `vercel.json` (if needed)
- `next.config.ts` (updated if needed)
