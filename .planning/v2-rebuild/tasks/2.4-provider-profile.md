# Task 2.4 — Provider Public Profile Page

## Goal
Create a public-facing provider profile page that shows their info, services, availability, and reviews.

## Prerequisites
- Task 2.1 complete (profiles)
- Task 2.2 complete (services)
- Task 2.3 complete (availability)

## Context
When customers find a provider (via search or direct link), they see this profile page. It shows the provider's bio, services with prices, working hours, and reviews.

## Steps

### 1. Provider Profile Page
Create `app/provider/[id]/page.tsx`:
- Server component — fetch provider data by ID
- Fetch from Supabase: profile, services (active only), availability, reviews
- If provider not found, show 404

### 2. Page Sections

**Header:**
- Provider name, avatar (or initials placeholder), rating (stars), number of reviews
- Bio/description
- Location/area (text, not exact address for privacy)

**Services Section:**
- List of active services as cards
- Each shows: title, description, price, duration
- "Book This Service" button on each (links to `/booking/[providerId]?service=[serviceId]` — page doesn't exist yet, just set up the link)

**Availability Section:**
- Weekly schedule displayed as a simple table/grid
- Days with hours shown, unavailable days marked

**Reviews Section:**
- List of reviews with rating (stars), customer name, comment, date
- Provider's reply shown if exists
- "No reviews yet" empty state

### 3. Helper Components
Create these in `components/providers/`:
- `provider-card.tsx` — reusable provider card (name, rating, photo, location) — will be used in search results later
- `star-rating.tsx` — displays star rating (filled/empty stars using Lucide Star icon)

### 4. Design
- Dark theme throughout
- Gold stars for ratings
- Service cards with price prominently displayed
- Clean section separation
- Responsive: single column on mobile, wider layout on desktop

### 5. Verify
```bash
npm run build
```

## Output Files
- `app/provider/[id]/page.tsx`
- `components/providers/provider-card.tsx`
- `components/providers/star-rating.tsx`
