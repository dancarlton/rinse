# Task 3.3 — Provider List/Grid Alongside Map

## Goal
Create the provider list component that displays search results alongside the map.

## Prerequisites
- Task 3.1 complete (search page exists)
- Task 3.2 complete (API endpoint exists)
- Wave 2 complete (provider-card component exists)

## Steps

### 1. Provider List Component
Create `components/providers/provider-list.tsx` (client component):
- Receives array of providers as prop
- Displays providers in a scrollable vertical list
- Each provider shown using the `ProviderCard` component (from Task 2.4)
- Show: name, rating, distance (if available from API), top service + price
- Click card to navigate to `/provider/[id]`
- Loading skeleton state
- Empty state: "No providers found in this area. Try expanding your search radius."

### 2. Update Provider Card
Enhance `components/providers/provider-card.tsx` if needed:
- Add distance display (e.g., "2.3 km away")
- Add starting price display (e.g., "From $30")
- Hover effect (subtle border glow or lift)

### 3. Integrate with Search Page
Update `app/search/page.tsx`:
- Wire up the provider list to the search results
- When a provider card is hovered, highlight the corresponding map marker (optional, nice-to-have)
- Ensure the list scrolls independently from the map

### 4. Design
- Cards in a single column, scrollable
- Each card: provider name, star rating, distance, starting price, "View" button
- Dark cards with subtle borders
- Gold accent on hover or featured providers
- Smooth scrolling

### 5. Verify
```bash
npm run build
```

## Output Files
- `components/providers/provider-list.tsx`
- `components/providers/provider-card.tsx` (updated if needed)
- `app/search/page.tsx` (updated with list integration)
