# Task 6.1 — Responsive Design + Mobile Optimization

## Goal
Ensure all pages are fully responsive and optimized for mobile devices.

## Prerequisites
- Waves 1-5 complete

## Steps

### 1. Audit All Pages
Go through each page and ensure responsive breakpoints work:
- Landing page (`app/page.tsx`)
- Login/Register pages
- Search/Map page
- Provider profile page
- Booking flow
- Provider dashboard (all sub-pages)
- Customer dashboard
- Settings page

### 2. Key Responsive Patterns
- **Navbar**: Hamburger menu on mobile (already done in 1.5)
- **Search page**: Stack map above list on mobile (map takes ~40vh)
- **Dashboard sidebars**: Collapse to top tabs or hamburger on mobile
- **Cards**: Single column on mobile, grid on desktop
- **Forms**: Full width inputs on mobile
- **Tables**: Horizontal scroll or card layout on mobile

### 3. Touch Optimization
- Minimum 44px touch targets for buttons
- Adequate spacing between interactive elements
- No hover-only interactions (provide tap alternatives)

### 4. Mobile-Specific Improvements
- Add `viewport` meta tag if not present
- Ensure images use `next/image` where appropriate for optimization
- Test font sizes are readable on small screens (min 14px body text)

### 5. Tailwind Responsive Classes
Use Tailwind's responsive prefixes consistently:
- `sm:` (640px+)
- `md:` (768px+)
- `lg:` (1024px+)
- `xl:` (1280px+)

### 6. Verify
```bash
npm run build
```

## Output Files
- Multiple page/component files updated with responsive improvements
