# Task 5.1 — Provider Dashboard (Overview with Bookings, Earnings, Reviews)

## Goal
Create the provider dashboard overview page showing key metrics and recent activity.

## Prerequisites
- Wave 4 complete (bookings, payments exist)

## Steps

### 1. Provider Dashboard Overview
Update `app/dashboard/provider/page.tsx`:
- Server component — fetch aggregated data
- Auth guard: redirect to `/login` if not authenticated, redirect to `/onboarding` if not onboarded

### 2. Dashboard Sections

**Stats Cards Row:**
- Total bookings (all time)
- Earnings this month (sum of completed booking amounts minus platform fee)
- Average rating
- Upcoming bookings count
- Each stat in a card with icon and value

**Upcoming Bookings:**
- Next 5 upcoming confirmed/in-progress bookings
- Show: customer name, service, date/time
- Quick action: "Start Service" or "View Details"
- Link to full bookings page

**Recent Reviews:**
- Last 3 reviews
- Star rating, customer name, comment preview
- "Reply" button if no reply yet
- Link to all reviews

**Quick Links:**
- Manage Services
- Set Availability
- Stripe Payments
- View Profile (as customers see it)

### 3. Earnings Calculation
Create helper in `app/actions/bookings.ts` or a new file:
- `getProviderEarnings(providerId: string, period: 'week' | 'month' | 'all')` — sum completed booking amounts, subtract platform fee

### 4. Design
- Grid layout for stat cards (2x2 on mobile, 4 across on desktop)
- Cards with icons from Lucide (Calendar, DollarSign, Star, Clock)
- Gold accent on key numbers
- Clean section separation
- Dark cards with subtle borders

### 5. Verify
```bash
npm run build
```

## Output Files
- `app/dashboard/provider/page.tsx` (full dashboard overview)
