# Task 6.2 — Loading Skeletons, Error Boundaries, Toasts

## Goal
Add loading states, error handling, and user feedback throughout the app.

## Prerequisites
- Waves 1-5 complete

## Steps

### 1. Root Loading/Error
Create `app/loading.tsx`:
- Full-page loading spinner or skeleton
- Dark background, gold spinner/accent

Create `app/error.tsx`:
- Client component (required by Next.js)
- Shows error message with "Try Again" button
- Dark themed error card

Create `app/not-found.tsx`:
- Custom 404 page
- "Page not found" with link back to home
- Dark themed

### 2. Route-Specific Loading States
Create loading.tsx files for key routes:
- `app/dashboard/loading.tsx` — dashboard skeleton
- `app/search/loading.tsx` — map + list skeleton
- `app/provider/[id]/loading.tsx` — profile skeleton
- `app/booking/[providerId]/loading.tsx` — booking form skeleton

Each should show a skeleton approximation of the actual content.

### 3. Skeleton Components
Create `components/ui/skeleton.tsx`:
```tsx
import { cn } from "@/lib/utils"

function Skeleton({ className, ...props }: React.HTMLAttributes<HTMLDivElement>) {
  return (
    <div
      className={cn("animate-pulse rounded-md bg-muted", className)}
      {...props}
    />
  )
}

export { Skeleton }
```

### 4. Toast Setup (if not done in 5.4)
Ensure sonner or similar toast system is set up:
- `<Toaster />` in root layout
- Use throughout for: "Service saved", "Profile updated", "Booking confirmed", errors

### 5. Error Boundaries for Key Sections
Create `app/dashboard/error.tsx`
Create `app/search/error.tsx`

### 6. Verify
```bash
npm run build
```
