# Task 4.2 — Booking Flow (Service → DateTime → Location → Checkout)

## Goal
Create the multi-step booking wizard where customers select a service, choose date/time, enter location, and pay.

## Prerequisites
- Task 4.1 complete (Stripe Connect setup)
- Wave 2 complete (services, availability)
- Wave 3 complete (provider pages)

## Steps

### 1. Booking Server Actions
Create `app/actions/bookings.ts`:
```ts
'use server'
```
Actions:
- `createBooking(data)` — create a booking record with status 'pending'
- `createPaymentIntent(bookingId: string)` — create Stripe PaymentIntent with:
  - `amount` from the service price (in cents)
  - `application_fee_amount` = 15% of amount
  - `transfer_data.destination` = provider's Stripe account ID
  - Return client_secret for Stripe Elements
- `getBooking(id: string)` — fetch booking details
- `getProviderBookedSlots(providerId: string, date: string)` — check which time slots are already booked for a date

### 2. Booking Page
Create `app/booking/[providerId]/page.tsx`:
- Client component (needs interactive multi-step form)
- Fetch provider info, services, and availability on mount

### 3. Multi-Step Booking Wizard
Create `components/booking/booking-wizard.tsx` (client component):
Steps:
1. **Service Selection** — show provider's services, select one
2. **Date & Time** — calendar/date picker for date, time slot grid based on provider's availability
   - Show available time slots (filter out already-booked slots)
   - Time slots in 30-min increments within provider's hours for that day
3. **Location** — enter address where service will be performed
   - Text input for address
   - "Use My Location" button (browser Geolocation)
4. **Review & Pay** — summary of booking details, Stripe payment
   - Show: service name, price, date/time, location
   - Stripe Elements for card payment
   - "Confirm & Pay" button

### 4. Install Stripe React
```bash
npm install @stripe/stripe-js @stripe/react-stripe-js
```

### 5. Payment Component
Create `components/booking/payment-form.tsx` (client component):
- Load Stripe with `loadStripe(process.env.NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY!)`
- Use `Elements` provider and `PaymentElement`
- Handle payment confirmation
- On success, redirect to `/booking/[providerId]/success?booking=[bookingId]`

### 6. Date/Time Selection Component
Create `components/booking/datetime-picker.tsx` (client component):
- Simple date grid (next 14 days)
- Time slot buttons based on provider availability
- Highlight selected date/time in gold
- Gray out unavailable/booked slots

### 7. Design
- Step indicator at top (1 → 2 → 3 → 4)
- Each step in a card
- Gold accent on selected items and CTAs
- Clean summary card in review step
- Mobile-friendly vertical layout

### 8. Verify
```bash
npm run build
```

## Output Files

- `components/booking/booking-wizard.tsx`
- `components/booking/payment-form.tsx`
- `components/booking/datetime-picker.tsx`
