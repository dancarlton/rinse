# Task 2.3 — Availability Weekly Schedule

## Goal
Create an availability management page where providers set their weekly working hours (e.g., Monday 9am-5pm, Tuesday 10am-6pm).

## Prerequisites
- Task 2.1 complete (provider onboarding)
- Task 2.2's provider dashboard layout exists

## Context
Providers set recurring weekly availability. Each day can have a start and end time, or be marked as unavailable. Customers see these when booking.

## Steps

### 1. Availability Server Actions
Create `app/actions/availability.ts`:
```ts
'use server'
```
Actions:
- `getAvailability(providerId?: string)` — fetch availability slots for a provider
- `setAvailability(slots: Array<{day_of_week: number, start_time: string, end_time: string}>)` — upsert all availability slots (delete existing, insert new)
  - Accept the full week's schedule and replace it atomically
- `deleteAvailabilitySlot(id: string)` — remove a specific slot

### 2. Availability Page
Create `app/dashboard/provider/availability/page.tsx`:
- Server component — fetch current availability
- Auth guard

### 3. Availability Form Component
Create `app/dashboard/provider/availability/availability-form.tsx` (client component):
- Display all 7 days of the week (Sunday-Saturday)
- For each day:
  - Toggle: available/unavailable for that day
  - If available: start time and end time selectors
  - Time selectors: use `<select>` dropdowns with 30-minute increments (6:00 AM through 10:00 PM)
- "Save Schedule" button at bottom
- Pre-fill with existing availability data
- On save, call `setAvailability` with the full schedule

### 4. Design
- Each day as a row or card
- Day name on the left, toggle in middle, time selectors on right
- Unavailable days grayed out
- Gold save button
- Clean table/grid layout
- Use Lucide icons if helpful (Clock, Calendar)

### 5. Verify
```bash
npm run build
```

## Output Files
- `app/actions/availability.ts`
- `app/dashboard/provider/availability/page.tsx`
- `app/dashboard/provider/availability/availability-form.tsx`
