# Task 6.5 — Seed Script for Demo Data

## Goal
Create a seed script that populates the database with realistic demo data for testing and demos.

## Prerequisites
- Schema migration (Task 1.3) applied to database

## Steps

### 1. Seed Script
Create `scripts/seed.ts`:

Using Supabase JS client with service role key (to bypass RLS):
```ts
import { createClient } from '@supabase/supabase-js'

const supabase = createClient(
  process.env.NEXT_PUBLIC_SUPABASE_URL!,
  process.env.SUPABASE_SERVICE_ROLE_KEY!
)
```

### 2. Demo Data

**Providers (5-6):**
- Various detailing businesses with realistic names, bios, locations around a metro area
- Names like: "Mike's Mobile Detail", "Pristine Auto Spa", "Clean Machine Detailing"
- Each with different service offerings and price points
- Ratings between 4.0-5.0 with varying num_ratings
- Locations spread around (use coordinates of a real metro area, e.g., LA or Miami)

**Services (3-4 per provider):**
- Basic Wash ($30-40, 30-45 min)
- Interior Detail ($60-80, 60 min)
- Full Detail ($120-180, 120-180 min)
- Ceramic Coating ($200-400, 180-240 min)
- Express Wash ($20-25, 20 min)

**Availability (per provider):**
- Most providers: Mon-Fri 8am-6pm, Sat 9am-4pm
- Some variation between providers

**Reviews (2-5 per provider):**
- Realistic comments: "Great service!", "Very thorough cleaning", "My car looks brand new"
- Ratings 4-5 stars
- Some with provider replies

### 3. Script Execution
Add to `package.json`:
```json
{
  "scripts": {
    "seed": "npx tsx scripts/seed.ts"
  }
}
```

Install tsx if needed:
```bash
npm install -D tsx
```

### 4. Important Notes
- The seed script should be idempotent (can run multiple times without duplicating)
- Use `upsert` where possible
- Skip auth user creation (providers need to be real auth users for the app to work)
  - Instead, seed the profiles table with UUIDs, and note that these won't have auth accounts
  - OR: document that the seed script is for the profiles/services/reviews tables only
- Log progress as the script runs

### 5. Verify
```bash
npm run build
```
The script itself doesn't need to run successfully (no DB connection in dev without Doppler), but it should compile.

## Output Files
- `scripts/seed.ts`
- `package.json` (updated with seed script)
