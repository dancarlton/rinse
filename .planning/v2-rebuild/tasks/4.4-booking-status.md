# Task 4.4 — Booking Status Management (Provider Actions)

## Goal
Let providers manage their bookings: confirm, start, complete, or cancel them.

## Prerequisites
- Task 4.2 complete (booking flow, bookings action file exists)

## Steps

### 1. Extend Booking Actions
Update `app/actions/bookings.ts` — add:
- `updateBookingStatus(bookingId: string, status: BookingStatus)` — provider can change status
  - Validate status transitions: pending→confirmed, confirmed→in_progress, in_progress→completed, any→cancelled
  - Verify the current user is the booking's provider
- `getProviderBookings(status?: BookingStatus)` — fetch bookings for the current provider, optionally filtered by status
- `getCustomerBookings(status?: BookingStatus)` — fetch bookings for the current customer

### 2. Provider Bookings Page
Create `app/dashboard/provider/bookings/page.tsx`:
- Server component — fetch provider's bookings grouped by status
- Tabs or filters: All, Pending, Confirmed, In Progress, Completed, Cancelled
- Each booking card shows:
  - Customer name
  - Service name and price
  - Date and time
  - Location/address
  - Status badge (color-coded)
  - Action buttons based on current status:
    - Pending: "Confirm" (green) / "Decline" (red)
    - Confirmed: "Start Service" (blue)
    - In Progress: "Complete" (green)
    - Completed: no actions, show "Done" badge
    - Cancelled: no actions, show "Cancelled" badge

### 3. Booking Status Component
Create `components/booking/booking-status-badge.tsx`:
- Color-coded status badges:
  - Pending: yellow/amber
  - Confirmed: blue
  - In Progress: purple
  - Completed: green
  - Cancelled: red/gray
- Reusable across provider and customer dashboards

### 4. Design
- Booking cards with clear status indication
- Action buttons are prominent and color-coded
- Filter tabs at top
- Empty states per tab
- Dark theme with status colors providing visual interest
- Responsive cards

### 5. Verify
```bash
npm run build
```

## Output Files
- `app/actions/bookings.ts` (updated with new actions)
- `app/dashboard/provider/bookings/page.tsx`
- `components/booking/booking-status-badge.tsx`
