# Task 2.1 — Provider Onboarding Flow

## Goal
Create the provider onboarding flow where new providers set up their profile, add their location, bio, and phone number.

## Prerequisites
- Wave 1 complete (Next.js, Tailwind/shadcn, Supabase, Auth)

## Context
When a user registers with role "provider", they need to complete an onboarding flow before they can receive bookings. This page collects their business info and location.

## Steps

### 1. Profile Server Actions
Create `app/actions/profile.ts`:
```ts
'use server'
```
Actions needed:
- `getProfile()` — fetch current user's profile from Supabase
- `updateProfile(formData: FormData)` — update profile fields (full_name, bio, phone, address, location)
- `completeOnboarding(formData: FormData)` — same as updateProfile but also redirects to provider dashboard

For location, accept `latitude` and `longitude` from the form and update the geography column using raw SQL or Supabase's PostGIS support:
```sql
UPDATE profiles SET location = ST_SetSRID(ST_MakePoint(lng, lat), 4326)::geography WHERE id = $1
```
Use `supabase.rpc()` or raw query for the geography update since the JS client doesn't natively handle PostGIS types.

Alternatively, create a Supabase RPC function in a new migration file `supabase/migrations/002_update_location.sql`:
```sql
CREATE OR REPLACE FUNCTION update_profile_location(
  user_id uuid,
  lat double precision,
  lng double precision
) RETURNS void AS $$
BEGIN
  UPDATE profiles SET location = ST_SetSRID(ST_MakePoint(lng, lat), 4326)::geography
  WHERE id = user_id;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```

### 2. Onboarding Page
Create `app/onboarding/page.tsx`:
- Server component that checks auth — redirect to `/login` if not authenticated
- Check if user role is 'provider', redirect to `/dashboard` if already onboarded (has bio and address)
- Render a client component `OnboardingForm`

Create `app/onboarding/onboarding-form.tsx` (client component):
- Step-by-step or single-page form with:
  - Full name (pre-filled from auth)
  - Bio / business description (textarea)
  - Phone number
  - Service area / address (text input)
  - Location picker: Use a simple approach — text input for address + hidden lat/lng fields
    - For MVP: use browser Geolocation API with a "Use My Location" button
    - Show lat/lng values to confirm location was captured
- Submit button: "Complete Setup" (gold primary)
- On submit, call `completeOnboarding` action, redirect to `/dashboard/provider`

### 3. Design
- Dark theme card layout, centered
- Gold accent on primary CTA
- Use existing shadcn components: Card, Input, Label, Button
- Add a Textarea component if needed (simple styled textarea, no need for shadcn)

### 4. Add shadcn Textarea component
Create `components/ui/textarea.tsx`:
```tsx
import * as React from "react"
import { cn } from "@/lib/utils"

const Textarea = React.forwardRef<
  HTMLTextAreaElement,
  React.ComponentProps<"textarea">
>(({ className, ...props }, ref) => {
  return (
    <textarea
      className={cn(
        "flex min-h-[80px] w-full rounded-md border border-input bg-background px-3 py-2 text-sm ring-offset-background placeholder:text-muted-foreground focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2 disabled:cursor-not-allowed disabled:opacity-50",
        className
      )}
      ref={ref}
      {...props}
    />
  )
})
Textarea.displayName = "Textarea"

export { Textarea }
```

```bash
npm run build
```
